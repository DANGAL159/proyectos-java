Clases //Inicial mayuscula
    Plantillas para la creacion de objetos, en ella se definen las propiedades(atributos) y comportamientos(metodos) los cuales caracterizan a nuestros objetos en especifico. Facilitan la organizacion y abstraccion del codigo.

Metodos // en minusculas
    Son bloques de codigo que encapsulan acciones especificos que realiza el objeto que lo modifican o responden a eventos.

Atributos //en minusculas
    Son las caracteristicas que describen a nuestro objeto, con ellas gestionamos al objeto, y pueden ser de todo tipo hasta otros objetos o estructuras de datos.

Objetos //en minusculas
    Instancias, ejemplificaciones de nuestra clase. Permiten modelar entidades del mundo real siendo una herramienta que facilita la organizacion y el diseño del software.

Ejemplo
    Clase
        Gatos
    Atributos
        Color : string
        Edad : integer
        Peso : integer
        EstaVivo : boolean
    Metodos
        Maullar
    Objeto
        Gato felix
        Gato bengali

Abstraccion 
    Generalizar la informacion en un concepto y simplifica el sistema

Encapsulamiento
    Mantener el codigo con cierto grado de seguridad para que no sea accesible por cualquier parte del codigo, usaremos metodos para que sean accesibles. Getter/Setter y public, private

Herencia
    Permite que una clase herede metodos y atributos de otra, de una SUPERCLASE...SUBCLASE para facilitar el codigo, cuando se llaman igual algun metodo se aplica el @Override   si usan el mismo metodo es optimo que solo este en la clase SUPER.

Polimorfismo
    Un metodo puede reaccionar de distinta forma dependiento el objeto interaccione con el, dando un codigo mas reutilizable. Se convima con la herencia normalmente.


Palabra STATIC → el atributo/metodo de clase que estara disponible para todo el programa.

    Suelen ser moldes para otras clases usando HERENCIA, usan la palabra    extends
    Son de tipo PUBLIC/ PROTECTED, nunca private

    Una clase no puede heredar varias clases ABSTRACTAS

    Un metodo abstracto solo declara el nombre del metodo pero no tiene contenido dentro de el, cada SUBCLASE vera como lo utilizaran, necesitan al menos 1 sino solo seria una clase SUPER
